macOS上安装Composer

wiki  /  tools 2017-07-11

简明教程

环境需要: PHP 5.3.2+

将 composer.phar 下载到当前目录:

$ curl -sS https://getcomposer.org/installer | php

PHAR（PHP 压缩包）是一个压缩格式，可以在命令行下直接运行。

http://www.jianshu.com/p/c06a626157a1 当你下载了 composer.phar 后，可以将它放在目录中，但每次当你建立新目录时，你必須再复制一个副本到新目录中，这样比较麻烦。所以最佳做法是将它放到 usr/local/bin 目录中中，成为全域指令。

mv composer.phar /usr/local/bin/composer

使用时直接用composer即可。

机翻 Installation - Linux / Unix / OSX

Locally

Installing Composer locally is a matter of just running the installer in your project directory. See the Download page for instructions.

在本地安装Composer是在项目目录中运行安装程序的问题。有关说明，请参阅下载页面。

The installer will just check a few PHP settings and then download composer.phar to your working directory. This file is the Composer binary. It is a PHAR (PHP archive), which is an archive format for PHP which can be run on the command line, amongst other things.

安装程序将检查几个PHP设置，然后将composer.phar下载到您的工作目录。该文件是Composer二进制文件。它是一个PHAR（PHP归档），它是PHP的归档格式，可以在命令行中运行。

现在只需运行php composer.phar即可运行Composer。

您可以使用–install-dir选项将Composer安装到特定目录，另外（重新）使用–filename选项命名它。运行安装程序时按照下载页面说明添加以下参数：

php composer-setup.php --install-dir = bin --filename = composer

现在只需运行php bin / composer即可运行Composer。

Globally

您可以将Composer PHAR放置在任何地方。 如果将其放在作为PATH一部分的目录中，则可以在全局访问它。 在unixy系统上，您甚至可以使其可执行并调用它，而无需直接使用php解释器。

根据“下载”页面的说明运行安装程序后，您可以运行此操作将composer.phar移动到路径中的目录：

mv composer.phar /usr/local/bin/composer

注意：如果以上由于权限失败，您可能需要使用sudo再次运行它。 注意：在某些版本的OSX上，默认情况下，/ usr目录不存在。 如果收到错误“/ usr / local / bin / composer：没有这样的文件或目录”，则必须手动创建该目录，然后再继续执行：mkdir -p / usr / local / bin。 注意：有关更改路径的信息，请阅读维基百科文章和/或使用Google

现在只需运行composer就可以了,而不用php composer.phar。
